Build your gesture wall, step by step

By the end of this guide you'll have two walls in a corner, each lit by a projector, watched by two Kinect depth cameras, where several people can walk around and point at tiles to select them — no mouse, no touch.

How to use this guide

  • Work top to bottom. Each step has a checkbox — tick it when done.
  • Your progress is saved automatically (you can close the tab and come back).
  • Grey boxes are commands — click Copy, paste into the Terminal app, press Return.
  • Green boxes tell you what you should see if it worked.
Don't panic. You do not need to understand the code. Follow the steps in order and check the green "what you should see" boxes.

⏱️ Plan for ~2–3 hours the first time. Hardware mounting is the slow part.

Phase 2 · Physical layout

Set up the two walls (the corner)

Your two walls meet at a corner and act as one continuous surface. The vertical line where they meet is the seam. People stand in the open area (the "pocket") and point at either wall.

Why a corner? It puts two big targets within one arm-sweep, and the seam gives the two cameras a natural overlap to hand a cursor across.

Phase 2 · Physical layout

Mount & align the two projectors

Do this before calibrating cameras. The cameras get calibrated to the projected image — if you move a projector later you must re-calibrate that wall.

Phase 2 · Physical layout

Place the two Kinect cameras

Each Kinect sits at its wall, facing back at the people (like a webcam on a TV) — not aimed at the wall. They need to see faces and raised arms.

With depth cameras, two is enough. Each Kinect alone gives a full 3D pointing ray — the second is for covering both walls and seeing people when one camera is blocked.

Phase 3 · Connect cameras

Wire a Kinect to the Mac

Check the Mac actually sees it. Paste this:

system_profiler SPUSBDataType | grep -i -A8 -E "kinect|xbox|nui"
What you should see: lines mentioning Xbox NUI Sensor or Microsoft. If you see nothing, re-seat the USB, try another port/adapter, and check the power brick light.

Phase 4 · Install software

Install the base tools

Paste each command into Terminal and press Return. Wait for each to finish before the next.

1. Apple's command-line tools

xcode-select --install

A popup appears — click Install and wait. If it says "already installed", great, move on.

2. Homebrew (the macOS software installer)

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

It may ask for your Mac password (typing shows nothing — that's normal) and to press Return. At the end it may print 2 lines starting with echo — paste & run those too.

3. The build tools we need

brew update && brew install git cmake pkg-config node python libusb glfw3 jpeg-turbo
Apple Silicon check: run brew --prefix — it should print /opt/homebrew. If it prints /usr/local you're in the wrong (Intel) Homebrew; ask for help before continuing.

Phase 4 · Install software

Install the Kinect driver (libfreenect2)

This is the software that reads depth + color from the Kinect v2 on macOS.

1. Download & build it

cd ~ && git clone https://github.com/OpenKinect/libfreenect2.git cd libfreenect2 && mkdir -p build && cd build cmake .. -DCMAKE_INSTALL_PREFIX="$HOME/.local" -DCMAKE_BUILD_TYPE=Release cmake --build . -j"$(sysctl -n hw.ncpu)" cmake --install .

2. Tell your Mac where it lives

echo 'export PKG_CONFIG_PATH="$HOME/.local/lib/pkgconfig:$PKG_CONFIG_PATH"' >> ~/.zshrc echo 'export DYLD_LIBRARY_PATH="$HOME/.local/lib:$DYLD_LIBRARY_PATH"' >> ~/.zshrc source ~/.zshrc

3. Prove the Kinect works

With one Kinect plugged in & powered, run the built-in test viewer:

cd ~/libfreenect2/build && ./bin/Protonect
What you should see: windows showing a live color image and a depth image (a colorful "heat-map" of the room). Wave your hand — the depth view changes. Close the window or press Esc to stop.
No device / no frames? Re-check USB 3 (Slide 6), avoid hubs, and try LIBUSB_DEBUG=3 ./bin/Protonect to see why. This is the #1 place people get stuck — it's almost always the USB-3 path or power.

Phase 4 · Install software

Build the Kinect "bridge"

The bridge is a tiny program that feeds the Kinect's depth + color into our app.

bash native/build_kinect_v2.sh
What you should see: it finishes with no error, and the file bin/kinect-v2-bridge now exists. Check with:
ls -l bin/kinect-v2-bridge
If it says "freenect2 not found": your library paths from Slide 8 step 2 aren't loaded. Close and reopen Terminal (or re-run the source ~/.zshrc line), then try again.
What's happening: the bridge uses libfreenect2 to grab a color image and a depth image that are pixel-aligned, so for any body joint we see, we instantly know how far away it is — that's what lets us build a 3D pointing ray.

Phase 5 · Configure

Create your room.json

This one file describes your room: the two walls, their tile grids, and the two Kinects. Start from the example:

cp room.example.depth.json room.json

What to edit (open room.json in any text editor)

SettingWhat it means
walls.A.gridHow many tiles on Wall A, e.g. {"rows":2,"cols":3} = 6 tiles. Same for B.
cameras.cam0.deviceWhich Kinect is camera 0 (usually 0). The second is 1.
server.num_posesMax people tracked at once. Start with 4.
calibration / plane / extrinsicLeave these for now — the next two slides fill them in by calibrating.
Don't hand-edit the numbers. The wall positions and camera angles get measured automatically in the next two slides — you don't type those by hand.

Phase 6 · Calibrate

Teach it where the walls are

Calibration measures the exact 3D position of each wall, as seen by each Kinect. Do this for each camera/wall pair.

Run it for camera 0 and Wall A (the tool will tell you which corner to point at):

.venv/bin/python -m gesturewall.calibrate --config room.json --camera cam0 --wall A

Now camera 0 and Wall B, then camera 1 for both (only the walls each camera can see):

.venv/bin/python -m gesturewall.calibrate --config room.json --camera cam0 --wall B .venv/bin/python -m gesturewall.calibrate --config room.json --camera cam1 --wall A .venv/bin/python -m gesturewall.calibrate --config room.json --camera cam1 --wall B
What you should see: after the 4th corner it prints calibration saved and writes the numbers into room.json.

Phase 6 · Calibrate

Line up the two cameras (shared room frame)

The two Kinects each have their own idea of "where things are." This step puts them in one shared map of the room, so a person seen by both cameras is understood as one person, and the seam handoff works.

Pick one camera as the reference (cam0) and register the other to it:

.venv/bin/python -m gesturewall.calibrate --config room.json --floor cam0 .venv/bin/python -m gesturewall.calibrate --config room.json --floor cam1
What you should see: it prints that it saved the camera positions (extrinsic) into room.json.

Phase 7 · Launch

Turn it on 🚀

1. Start the server (leave this Terminal running)

.venv/bin/python -m gesturewall.server --config room.json
What you should see: messages that it started, opened the cameras, and is serving on http://localhost:8000 + websocket 8770. Leave this window open.

2. Open one wall per projector

Open a browser window on projector 1 and go to:

http://localhost:8000/wall.html?wall=A&server=ws://localhost:8770

Drag it onto projector 1's screen, press f for fullscreen. Then on projector 2:

http://localhost:8000/wall.html?wall=B&server=ws://localhost:8770
What you should see: each wall shows its grid of tiles and a small "connected" indicator. Walk in and raise a hand — a colored dot (cursor) appears.

Reference

When something's wrong

SymptomMost likely fix
Mac doesn't see the KinectPower brick lit? Real USB-3 path (no cheap hub)? Re-run the system_profiler check (Slide 6).
Protonect opens but no frames / freezesUSB-3 bandwidth — plug Kinect directly, unplug other USB devices, try another USB-C adapter. LIBUSB_DEBUG=3 ./bin/Protonect to see why.
Bridge build: "freenect2 not found"Reopen Terminal (load the paths from Slide 8 step 2) and rebuild.
Server starts but cursors never appearAre people in the camera's range (0.5–4.5 m)? Is the wall calibrated for that camera? Is a hand raised above the shoulder?
Pointing is consistently off to one sideRe-calibrate that camera→wall pair; make sure projector hasn't moved since calibrating.
Two people swap cursors when they crossKnown limit with 2 cameras — give them more space, or add a third camera angle later.
Low frame rate / laggyClose other camera apps; the CPU depth pipeline is heavy — lower num_poses.
Browser won't open the camera (single-wall browser app)That's the in-browser app, not this server path. For the server path the cameras are opened by Python via libfreenect2.
