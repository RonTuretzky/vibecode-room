Microphone blocked on this connection

This page is served over plain http://, and this browser only allows microphone access on secure (HTTPS) origins.

  1. Android Chrome: open chrome://flags/#unsafely-treat-insecure-origin-as-secure, add to the list, set the flag to Enabled, and relaunch Chrome.
  2. iOS Safari: there is no override — getUserMedia requires HTTPS (or localhost). Use an Android phone or a desktop browser for the demo, or put the server behind a TLS proxy.
  3. Desktop Chrome/Firefox: open the page via http://localhost:<port>/mic.html on the server machine, or apply the same Chrome flag as Android.
