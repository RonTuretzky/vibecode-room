(The old AC9.3 read-back-length item is gone — the destructive read-back gate is cut, ENG-A-08.)

Per-process pause — deterministic, not NL

"Atlas, pause"   → callsign(Atlas) selects UPID → {type:'pause',  targetUPID:atlas}   → pauses ONLY Atlas
(after select) "Pause"                                                                  (state-gated to window)
"Bravo, resume"  → {type:'resume', targetUPID:bravo}
"pause the second one"  (no callsign, no window) → REJECTED at dispatch  (REQ-7/NG-2 — never NL)

This gives REQ-13's "steer A, pause B" a deterministic command path: spawn Atlas + Bravo, "Atlas, make it faster" steers A, "Bravo, pause" pauses B — A unaffected (AC13.1).

Verification

  • consent-content test — first announcement contains "Only transcripts are saved" AND names "mute" (RBG: drop the sentence → AC1.1 fails).
  • mute/unmute-word test — "mute"/"unmute" are the wired mute/unmute keywords (via Cue) (RBG: wire a different word → control no longer toggles → fails).
  • mute-announce / halt-announce tests — earcon + one-word "Muted"; E5 + ≤15-word halt TTS (RBG each: earcon-only → fails).
  • ambient-silence test (AC6.4) — ignored ambient speech emits no ack; only addressed outcomes get a distinct ack (RBG: make ambient pass emit a tick → fails).
  • per-process-pause routing test + e2e — "[callsign], pause" pauses that UPID only; NL "pause B" rejected at dispatch; "Atlas steer / Bravo pause" e2e (RBG: route NL pause → fails).
