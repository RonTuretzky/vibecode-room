Suggestion restraint is a set of env-tunable parameters — the labeled corpus is deferred

ENG-A-14REQ-3Simplified / deferred

Superseded / simplified. The prior decision engineered a fixed, owned, human-labeled replay corpus (ticket ENG-T-07) with a minimum size, label schema, multi-labeler protocol, dev/test split, and a hard restraint metric (recall ≥80% / ≤1 false-positive per 10 min, red-before-green). That formal corpus and metric are dropped for V0. ENG-T-07 is no longer a hard requirement.

The decision

For V0 there is no formal labeled corpus and no fixed restraint metric. Suggestion cadence / restraint is simply a set of env-tunable parameters with documented defaults (ties to ENG-A-01) — thresholds, cooldowns, and gates passed in via ENV, each with a default — that we tune by feel later from real-world UX. We do not stand up a labeling pipeline or assert a recall/FP number against a held-out test split.

Why simplified

Owning a human-labeled corpus (≥2 labelers, adjudication, inter-annotator agreement, leakage tests, a size floor) is heavy machinery whose only job was to make one number reproducible. The V0 posture — ship fast, make everything env-tunable, tune by feel — replaces that number with knobs. If suggestions are too chatty or too quiet, we turn a dial, not re-run a labeled eval.

What it looks like instead

Knob (ENV, documented default)Effect
SUGGESTION_MIN_CONFIDENCEhow confident the idea engine must be before it speaks
SUGGESTION_COOLDOWN_MSminimum gap between suggestions (restraint)
SUGGESTION_MAX_PER_WINDOWcap on suggestions in a rolling window

Defaults are documented; all are externally injected and tuned later by feel. No corpus, no metric, no red-before-green eval gate.

Verification

  • Knob-wiring test: each restraint parameter reads from ENV with its documented default and actually changes behavior (RBG: hard-code a threshold ignoring ENV → override test fails).
  • No-corpus assertion: there is no ENG-T-07 labeled corpus or held-out test-split eval gate in V0.
Note: a labeled corpus and a formal restraint metric can be revisited later if restraint-by-feel proves insufficient — but it is not V0 scope.
