The annotated replay corpus is a concrete, owned, versioned artifact

The decision

The restraint metric — recall ≥80% on "should-suggest", ≤1 false-positive / 10 min on "should-pass" (AC3.4) — is reproducible only against a fixed, owned, human-labeled corpus, not a vague "suite." It is engineered as ticket ENG-T-07 with a defined location, size floor, label schema, labeling protocol, dev/test split, and recorded red-before-green.

Why — the R16 finding

"§13 references an annotated replay suite but does not define where it lives, minimum duration, label schema, labeling protocol, train/test separation, representative negative audio, or how red-before-green evidence is recorded. Without that, the most important restraint metric is not reproducible."

The contract

AspectSpecification
Location / formartifacts/smithering/corpus/replay/ — transcript-observation JSONL + labels.json (NOT raw audio in the product; any seed .wav lives only in a dev-only store fenced outside the product, so §12's no-raw-audio guarantee holds). CORPUS.md records provenance + CHANGELOG.
Minimum size≥60 min "should-pass" (negative) audio (≥6 ten-minute windows for a meaningful ≤1/10-min FP rate) + ≥50 distinct "should-suggest" segments. Floors, not caps.
Label schema{segmentId, startMs, endMs, label:'should-suggest'|'should-pass', topic, rationale, speakerCount, conditions:['clean'|'overlap'|'noise'|'near-homophone']}
Labeling protocol≥2 independent labelers; third-party adjudication of disagreements; inter-annotator agreement recorded (Cohen's κ, target ≥0.7).
Representative negativestechnical jargon, build/test talk, 2-speaker overlap, silence, and near-homophones of every always-hot word (callsigns, Panop, Curtain, Daybreak, Abort).
Train/test separationCorpus is test-only. Knobs tuned on a separate dev split; the held-out test split runs only for the AC3.4 number. A leakage test asserts splits are disjoint by segmentId.
Red-before-greenShuffle the ground-truth labels → recall/FP collapse → suite fails (proves discrimination). Red + green runs archived under artifacts/smithering/reports/.
OwnershipEngineering team; corpus version + measured (recall, FP) recorded per acceptance run.

How it runs

[real audio, dev-only] → [Deepgram, recorded ONCE] → corpus/replay/*.jsonl + labels.json
                                                            │
   dev split  → tune suggestion knobs/thresholds            │  (never the test split)
   test split → decision loop @ temp-0 → fired vs observe.pass → compare to labels
                                                            │
                       recall ≥80%  AND  ≤1 FP / 10 min  →  AC3.4 PASS
   RBG: shuffle labels → recall collapses → FAIL (the suite can actually go red)
This makes AC3.4 — the product's single most important "restraint" number — a reproducible, regression-tracked measurement rather than an assertion.
