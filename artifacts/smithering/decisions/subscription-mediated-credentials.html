Hot-loop model calls route through Smithers subscriptions — no raw keys, redacted everywhere

ENG-A-15PRD §6Resolves R15Probe: P-LLM + A-LLM-SUB

The decision

PRD §6 is binding: all model calls route through Smithers subscriptions, never a raw API key. The Cue hot-loop DecisionLLM obtains model access from the same subscription path the per-process planner uses, via one SubscriptionCredentialProvider — so Cue holds no key. ASR/TTS credentials (audio services, not model calls) come through the same provider and obey the same rule. The TraceProcessor runs a fail-closed redaction filter over every log line and probe report. Default hot-loop model = Claude Haiku-4.5 via subscription (compliant by construction); Cerebras is a fallback only if subscription-routable or the PRD amends §6.

Why — the R15 finding

"§2/§16 define a Cue hot-loop DecisionLLM using Cerebras/Haiku/OpenAI-compatible providers but do not state these calls are mediated by Smithers subscriptions, nor define credential handling or log-redaction tests. This leaves an undeclared secrets and dependency path."

One credential path

SubscriptionCredentialProvider  (the ONLY place a credential is obtained, at construction)
   ├── DecisionLLM (hot loop)      ─┐
   ├── per-process planner          ├─ model access via Smithers SUBSCRIPTION (no raw key in source/env/Cue)
   ├── ASR (Deepgram)              ─┤
   └── TTS (selected provider)     ─┘
                                       │
   TraceProcessor.redact(LogEvent.meta, probeReport)   ── fail-closed ──▶  «redacted»
       matches bearer tokens, sk-…, Deepgram/ElevenLabs key shapes, JWTs, Authorization: headers
       unrecognized-but-secret-shaped → redacted, not passed

Verification

  • subscription-path test — DecisionLLM obtains access via the provider and fails construction if handed a raw key (RBG: inject OPENAI_API_KEY path → fails).
  • secret-redaction test — a fake token in meta/probe report emits «redacted» and the raw value appears nowhere (RBG: disable filter → leaks → fails).
  • per-probe redaction — P-ASR/P-TTS/P-LLM each assert zero key-shaped strings in their own traces.
  • whole-session secret-scan — grep the full trace/log/report tree → zero key-shaped strings (RBG: plant a key → found → fails).
  • A-LLM-SUB (blocking) — confirm a §6-compliant hot-loop model (subscription-routed, ~100 ms, temp-0) exists; if not, a binding PRD-§6 conflict to resolve at the gate.
Surfaced to the gate: if the only model meeting the ~100 ms hot-loop budget cannot be reached through Smithers subscriptions, that is a PRD-§6 conflict — amend §6 with secure credential handling, or accept a slower compliant model. Not engineered around.
