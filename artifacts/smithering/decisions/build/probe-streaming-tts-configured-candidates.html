P-TTS configured candidate handling

BUILD   probe-streaming-tts  ·  2026-06-15

Decision

Benchmark every 2026 TTS candidate whose credentials are configured in the host environment, record unconfigured candidates by provider name and env var name only, and leave the probe verdict red when no measured provider meets the 200 ms first-audio-byte budget or when the candidate benchmark is incomplete. Record a fastest measured provider separately from a selected provider. Do not synthesize placeholder wins or placeholder pre-cache clips for unavailable vendors.

Facts From The Probe

Configured provider:
OpenAI /v1/audio/speech, model gpt-4o-mini-tts, voice coral

Final 200 ms run measured:
first audio byte = 465 ms
budget = 200 ms
selected provider = none

Unconfigured:
ElevenLabs Flash family: ELEVENLABS_API_KEY or XI_API_KEY
Cartesia Sonic: CARTESIA_API_KEY
PlayHT 3.0 Turbo: PLAYHT_API_KEY and PLAYHT_USER_ID

Alternatives

Mark the probe green with OpenAI as the winner despite missing the budget. Rejected. This would unblock dependent REQ-10 work on an unproven round-trip budget.
Fail immediately when any vendor credential is absent. Rejected. The probe can still gather useful real evidence from configured candidates while clearly recording the missing families.
Measure configured providers and keep the verdict red if none pass. Chosen. This preserves probe precedence: dependents stay unscheduled until a real provider passes.
Fail closed on selected-provider pre-cache when no provider is selected. Chosen. The artifact records the blocker instead of treating synthetic bytes as evidence for real static clips.
