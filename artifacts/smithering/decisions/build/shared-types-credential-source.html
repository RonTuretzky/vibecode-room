CredentialSource is provenance only, not a credential provider

shared-types-contractENG-T-01

Problem

The ticket requires a CredentialSource export, while the V0 scope cut removes the prior credential-provider subsystem. The contract still needs a typed way to describe where access comes from without carrying any secret value.

Decision

CredentialSource records non-secret provenance only: host subscriptions for model access, or a redacted environment source for audio providers. It contains no value field and no raw credential string.

type CredentialSource =
  | { kind: "host-subscription"; provider: "openai-codex" | "anthropic-claude"; command: string }
  | { kind: "environment"; provider: "deepgram" | "tts"; variable: string; redacted: true }

Alternatives

AlternativeVerdictReason
Rebuild the old credential-provider abstractionRejectedThe V0 scope cut explicitly removes that subsystem.
Omit CredentialSourceRejectedThe ticket explicitly requires the shared contract export.
Non-secret provenance unionChosenSatisfies the contract while preserving the no-raw-key rule.

Verification

The zod schema is strict and has no field that can hold the secret value. The bundle secret scan reports zero key-shaped strings.
