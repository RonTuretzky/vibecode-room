Credential-shaped env values and trace identifiers fail closed

subscription-credentials-redactionSEC-1

Problem

Follow-up review found that model credential construction rejected canonical provider key variables but accepted provider-shaped values under alternate env names. The same review found that neutral trace identifier fields could persist token-shaped values even though metadata was already redacted.

Decision

Scan every supplied model env value with the same fail-closed secret detector used for emitted trace data. If any env value is credential-shaped, reject construction and keep model access on host Codex/Claude subscriptions. Trace emission now sanitizes event, sessionId, correlationId, and upid as well as metadata. When an event name itself is unsafe, it is replaced with the valid event name redacted.secret. 

input trace:  { event: "observe.provider-token", correlationId: "provider-token" }
output trace: { event: "redacted.secret", correlationId: "redacted marker" }
