Callsign phonetic guard

BUILD   callsigns-and-collision-guard  ·  2026-06-15

Decision

Use a local deterministic phonetic implementation in src/routing/callsigns.ts: a compact double-Metaphone-compatible code plus a normalized phoneme string for Levenshtein distance. The V0 pool is restricted to coined multi-syllable callsigns and the allocator validates every proposed or sequential callsign against active callsigns and the fixed wake/mute/unmute/panic/stop words.

Alternatives

Add a package dependency for phonetics. Rejected. P-PHONETIC is explicitly non-blocking and the repo has no existing phonetic dependency; adding one would widen supply-chain and install surface for a small deterministic guard.
Use edit distance over spelling only. Rejected. That misses same-sound variants and does not satisfy D-DD-05's Metaphone code collision requirement.
Keep the guard local and snapshot stable outputs. Chosen. The codes are reproducible in unit tests, swappable later, and every route stays in deterministic code rather than LLM judgment.
