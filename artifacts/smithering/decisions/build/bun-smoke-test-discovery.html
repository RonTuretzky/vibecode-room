Bun smoke test discovery shim

BUILD   walking-skeleton-smoke  ·  2026-06-14

Decision

Keep the required smoke source at test/smoke/spine-skeleton.smoke.ts, and add a tiny discovery shim at test/smoke/spine-skeleton.smoke.ts.test.ts that imports it. The ticket's exact command, bun test test/smoke/spine-skeleton.smoke.ts, then selects the shim because its valid Bun test filename contains the requested path pattern, while the required file remains the actual smoke test source.

Why

Bun 1.3.14 only discovers test files with supported suffixes such as .test.ts or .spec.ts when a bare pattern is passed. The required filename ends in .smoke.ts, so the exact command initially reported that no test file matched. Prefixing the path with ./ works locally, but the ticket's verification command is explicit and should pass as written.

Alternatives

Rename the required file to a Bun-native suffix. Rejected. It would satisfy discovery but violate the ticket's named path.
Require callers to use ./test/smoke/spine-skeleton.smoke.ts. Rejected. The verification command is fixed in the ticket.
Add a one-line Bun discovery shim. Chosen. It preserves the required file and makes the exact command executable under Bun's discovery rules.

Verification

bun test test/smoke/spine-skeleton.smoke.ts
bun test
bunx tsc --noEmit

All three commands pass after the shim. The red-before-green log for the smoke gate breaks the matcher, not the shim, so the evidence proves the real smoke assertion is capable of failing.
