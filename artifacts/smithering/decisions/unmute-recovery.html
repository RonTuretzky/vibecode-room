Voice-unmute is the sole operational unmute; emergency stop is kill-all only

ENG-A-13REQ-2 / REQ-14 / D1Resolves R10Probe: P-SPOTTER (now blocking)

The decision

The local on-device "Daybreak" spotter is the one and only operational unmute path, so it is on the critical path for REQ-2/D1 and P-SPOTTER is promoted to a blocking P0 probe. The REQ-14 non-voice emergency control stays exactly what the PRD says — a kill-all that halts every process and stops listening, ending the session — and is never reused as an in-session unmute. When the spotter is unavailable, the spotter-down recovery is: emergency kill-all → a fresh session restarts unmuted (with the §12 consent re-spoken). A teardown + clean restart, not a resume.

Why — the R10 finding

"§5.4/§11/§17 say a disabled/failing spotter degrades to the non-voice emergency control and even test that it 'unmutes.' REQ-14 defines that control as emergency-only kill-all that exposes only kill-all — not an unmute/resume path. This weakens D1 and makes P-SPOTTER incorrectly non-blocking for voice-unmute."

The two states reconciled

NORMAL:   "Curtain" → hard-mute cloud ASR (≤500 ms) + earcon + one-word "Muted"
              while muted: ONLY the local spotter is alive (no network, no transcript, no observations)
          "Daybreak" (spotter) → mute.released → cloud ASR resumes        ← the ONLY operational unmute

SPOTTER DOWN (host lacks it):  there is NO in-session unmute.
          REQ-14 emergency control → KILL-ALL + stop listening → SESSION ENDS
              → operator starts a fresh session → begins UNMUTED, consent re-spoken
          (REQ-14 stays kill-all only; AC14.2 preserved; D1's voice-only model intact)

Verification

  • no-unmute-verb test — the emergency control exposes no resume/unmute path (RBG: add an unmute route → AC14.2 scope test fails).
  • unmute-recovery test/e2e — with the spotter disabled, there is no in-session unmute; triggering REQ-14 performs kill-all + stop-listening (session ends); a fresh session starts unmuted with consent re-spoken (RBG: wire the emergency control to resume streaming → fails).
  • mute-announce test — "Curtain" emits earcon + one-word "Muted" (AC2.4).
  • P-SPOTTER (blocking) — detects "Daybreak", rejects near-homophones, emits only mute.released, no transcript, fully local.
Surfaced to the gate: a one-line PRD clarification — REQ-14 is the documented escape from a stuck mute only as "kill-all + restart," never "unmute in place."
