The shell-command classifier — keeping Safe mode autonomous

ENG-A-12REQ-11 AC11.1/AC11.2Resolves R9 (critical)Probe: P-SHELL-PARSE

The decision

The in-run safety hook (ENG-A-08) must not gate the shell class wholesale. A coding agent runs harmless shell constantly (ls, tests, typechecks, git inspection); gating all of it turns Safe into Explicit and violates AC11.1 ("the default posture runs to completion without per-step approval"). Instead, a deterministic, parse-based, code-only classifier (safety/shell-classifier.ts) sub-classifies each shell call into read-safe (ungated) vs. mutating/unknown (gated), deny-by-default. No LLM — same command in, same ShellVerdict out, replayable.

Why — the R9 finding

"§8 lists shell as a gated class while AC11.1 requires autonomy. If all shell is gated, Safe = Explicit (violates AC11.1); if selectively allowed, the doc defines no deterministic allow/deny policy or tests for safe vs destructive vs unknown vs compound vs redirect vs injection."

The algorithm (fully deterministic)

parse(cmd)  ──▶ split on && || ; |  and newlines into SIMPLE COMMANDS   (via shell-quote → P-SHELL-PARSE)
   for each simple command, classify by argv[0] + flags + redirections:
      read-safe  : ls pwd cat head tail wc file stat echo printf env which type date tree sort uniq cut
                   grep/rg, find (NO -delete/-exec), sed/awk (NO in-place/write), diff cmp,
                   git status|diff|log|show|branch|remote|rev-parse|describe|blame|ls-files|config --get,
                   bun test, tsc --noEmit, node/bun --version
      mutating   : rm rmdir unlink shred dd mkfs truncate chmod -R chown -R kill pkill reboot shutdown,
                   mv/cp over existing, git push|reset --hard|clean -fd|checkout -- , bun/npm install|publish,
                   docker rm|rmi|system prune, kubectl delete, terraform apply|destroy, psql/mysql -e 'DROP|...'
      redirect   : > or >> to a real path  → mutating   ( >/dev/null, 2>&1 → ignored )
      injection  : $(...) ` ` eval exec source/. <(...) here-doc|sh  OR any unparseable token → unknown
      unknown    : argv[0] in neither list → unknown
   COMPOUND VERDICT = most dangerous of the parts   (read-safe < mutating ≤ unknown for gating)
   any part mutating/unknown  →  GATE the whole command

Deny-by-default is the backstop: the allowlist enumerates only the safe programs; everything else (incl. obfuscated/opaque) is gated. The destructive list is a fast-path, not the authority.

Alternatives considered

Gate all shellRejectedSafe becomes Explicit → violates AC11.1.
Allow all shell, gate by NL verb upstreamRejectedThe R1/R9 failure — misses the real tool call; unsafe.

Parse → per-part verdict → most-dangerous compound, deny-by-defaultChosenAutonomous for harmless shell, gated for destructive/unknown; deterministic + testable.

Verification (each is an RBG test)

  • safe-shell (ls/git status/bun test/grep/find → ungated; RBG: move ls to mutating → AC11.1 e2e fails)
  • destructive-shell (rm/push --force/dd/truncate/kubectl delete → gated; RBG: drop rm → runs ungated → fails)
  • unknown-command (gated; RBG: default unknown→allow → fails) · compound (RBG: classify first part only → fails)
  • redirect (overwrite gated, /dev/null ungated) · injection ($()/backtick/eval/<() → gated)
  • fuzz (random/base64/here-doc/nested → never silently read-safe) · determinism (same in → same verdict)
  • e2e: one Safe-mode run does bun test+git status+grep with zero prompts AND is held on rm -rf/git push --force — proving Safe ≠ Explicit.
  • P-SHELL-PARSE: the parser splits compound/redirect/substitution as assumed (unparseable → unknown).
