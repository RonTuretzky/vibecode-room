Execution modes: Safe (default) / Explicit / Dangerous

ENG-A-11REQ-11 AC11.4Resolves R7 (modes)

The finding (R7, modes half)

"AC11.4 requires both dangerous mode and fully-explicit mode to be opt-in by voice and off by default. … lines mention only Dangerous mode, not a fully-explicit mode or voice command path."

AC11.4 requires two opt-in postures beyond the Safe default. The prior doc named only Dangerous. Both are now first-class, each reachable only by a warned, state-gated voice command.

The three modes

ModeThe in-run PreToolUse hook (§8)…Opt-inDefault
safegates mutating / unknown tool classes (deny-by-default)—ON (default)
explicit (fully-explicit)gates every action — per-step approvalvoice, warnedOFF
dangerousbypasses the gate entirelyvoice, warnedOFF

Properties (all enforced in code)

  • Default is safe for every fresh process (unit test: fresh process is safe; RBG: default to dangerous → fails).
  • Voice opt-in only via the state-gated mode commands ("Panop, dangerous mode" / "Panop, explicit mode" / "Panop, safe mode"), each requiring a spoken warning read-back + affirmative confirm before taking effect.
  • Session-only: a non-default mode resets to safe at session end (D-DD-19) — a persistent dangerous mode surviving restarts is an accident waiting to happen (unit test: persist dangerous across sessions → fails).

Action contract

Each mode command maps to exactly one handler emitting {type:'setMode', payload:{mode}}; ExecutionMode = 'safe' | 'explicit' | 'dangerous' is part of the shared types.ts contract, so the command vocabulary, the action schema, and the handlers all cover every documented command (the R7 traceability fix).

Verification

  • Unit: mode-default (safe); explicit-mode (every action gated); dangerous-mode (bypass only after warning + confirm; resets at session end); mode-command (each flips mode + requires warning + confirm).
  • E2e: switch to explicit → a non-destructive action is gated; switch to dangerous (with warning) → a destructive act proceeds without a gate; new session → back to safe.
