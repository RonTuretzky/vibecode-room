Mute-while-listening: an on-device unmute spotter

D-DD-22 · P0-adjacent   Panopticon V0 — final pass  ·  docs/planning/02-design.md §12

The contradiction caught in this pass

REQ-2 AC2.3: while muted, no observations are produced and no audio streams to the ASR provider.  vs.  D1: voice is the sole operational modality.  →  If nothing is listening while muted, how does the room unmute hands-free by saying "Daybreak"? The draft did not say. This is a genuine architectural gap, not a wording nit.

Decision

"Curtain" hard-mutes the cloud path (ASR stream stops ≤500 ms, E2 → mute tone, zero transcript observations). A minimal on-device keyword spotter stays active while muted. It runs fully local (no network, persists nothing), matches exactly one keyword — "Daybreak" — and emits exactly one event, mute.released. It cannot transcribe, route, suggest, or steer. If the spotter is unavailable on a host, unmute degrades to the non-voice emergency control (REQ-14), so the room is never trapped muted.

Why this preserves AC2.3 / NG-6

The spotter is not "the ASR." It produces no observations (the thing AC2.3 forbids) and writes no raw audio or transcript (NG-6 / AC1.3). Its only output is a single control event that flips the mute state back. So "no observations while muted" holds literally — the cloud path that would produce observations is closed for the entire interval.

Alternatives

A. Unmute only via the non-voice emergency control. Rejected as the primary path: turns a routine action into an emergency-only one and dents the hands-free promise (D1). Kept as the degraded fallback.
B. Keep the cloud ASR running but "ignore" everything except the unmute word. Rejected: audio still leaves the room and observations still exist — violates AC2.3 and NG-6 outright.
C. Local single-keyword spotter (chosen). Hands-free, nothing leaves the room, AC2.3/NG-6 intact, graceful degradation. Chosen.

Observability

New dependency surfaced to the gate

The on-device spotter is a third-party/model dependency that needs its own probe: matches "Daybreak", rejects near-homophones, emits no transcript. Listed in §15 of the design doc.
