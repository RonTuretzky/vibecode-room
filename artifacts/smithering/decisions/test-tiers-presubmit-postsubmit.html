The rule that makes an optimistic subset legitimate

An agent may gate the merge on only a subset of a ticket's checks iff the postsubmit tier runs every check, so nothing is permanently skipped. The pre-merge subset is the cheapest set that honestly proves the deterministic invariants; the slow, real-world, non-deterministic checks move to postsubmit where they belong — never deleted.

Partition rationale (cheapest method that actually proves it — matrix rule 2)

  • Deterministic invariant / logic / contract → a headless bun test is the honest gate, so it sits pre-merge (priority ladder, dispatch invariant, 15-word guard, collision guard, trace schema, shell classifier, hook-intercept on doubles).
  • A real-world property (measured latency, recall on a corpus, durability across a real restart, no-screen hands-free, real third-party behavior) → a mocked unit cannot prove it, so it sits postsubmit / pre-build (AC10.1 earcon ≤300 ms, AC10.2 p50/p95, AC3.4 recall/FP, AC15.3 recovery, AC5.1 no-screen, every P-*).
  • The matrix's AND (unit and e2e) is preserved across tiers: the unit half gates the merge, the e2e half gates the integration on main. Deleting either tier must still leave us fairly confident — exactly the bar.

Alternatives

Run the full matrix (incl. live e2e/eval/benchmarks) pre-merge. Rejected. Non-deterministic and credential/hardware-bound; would make every land flaky and slow, and can't run hermetically in the lane.
Pre-merge subset with no postsubmit backstop. Rejected. Silently skips the real-world gates forever — the matrix's whole point is that those properties are the ones a mock can't prove.
Deterministic subset pre-merge, full suite postsubmit, probes pre-build (chosen). Chosen. Honors the optimistic-subset rule, keeps lands fast and deterministic, and guarantees every gate eventually runs against the real thing.
