The provider interface is the universal test boundary

ENG-A-02   Vibersyn V0 engineering  ·  how the 10×–100× bar is made achievable  ·  docs/planning/03-eng.md §2, §13.1

The problem

The core is non-deterministic (ASR + LLM) and hardware/network-dependent (mic, Deepgram, TTS, Smithers). The validation bar demands 10×–100× more tests than a human would write. That volume is impossible if every test needs a live mic, real API keys, and network round-trips: it would be slow, flaky, and expensive. We need a boundary where the entire inner system can be tested headless and deterministically.

Decision — CONFIRMED, kept

Confirmed and retained. Put a typed provider interface at every external boundary, and test everything inside that boundary with record-replay doubles. No component constructs a provider directly — providers are injected at construction. This is the mechanism that makes the Cue integration modular and trivial to mock (ENG-A-01) and that makes verification possible without live mics, keys, or network. 
interface ASRProvider     { stream(audio): AsyncIterable<TranscriptObservation> }   // real: Deepgram | double: JSONL replay
interface TTSProvider     { speak(text, opts?): Promise<ReadableStream> }            // real: probe-selected | double: noop
interface DecisionLLM     { decide(input): Promise<DecisionOutput> }                 // real: host Codex/Claude subscription, temp-0 | double: fixture

The inner test loop reads pre-recorded ASR output as JSONL, runs the decision LLM at temperature-0, and asserts shape/invariants ("≤3 MCQs", "≤15 words", "fires within budget") — never exact text. Same input → same output, every run.

Alternatives

Test against live APIs. Rejected: slow, flaky, key-dependent, non-deterministic — cannot reach 10×–100× volume.
Mock at the network layer (HTTP/WebSocket). Rejected: couples every test to wire formats; the provider interface is a cleaner, more stable seam.
Inject doubles at the provider interface (chosen). Chosen. Headless, deterministic, key-free inner loop; the real providers are proven separately by their probes (§15).

Verification

  • Unit: boundary-substitution test — every consumer runs with replay/noop doubles, zero network/mic.
  • Unit: architecture lint — no concrete-provider import outside providers/; RBG: import Deepgram directly in a consumer → lint test fails.
  • Unit: interface-conformance — the replay ASR yields the same TranscriptObservation shape the real one promises.
