AC6.4 — ignored ambient speech is SILENT; only addressed commands get an ack

ENG-A-10REQ-6 AC6.4REQ-9 restraintResolves R6 / R14

Reversed: a prior pass implemented AC6.4 "as written" so that every routed utterance — including ignored ambient observe.pass / route.pass — produced a distinct audible ack, with silence available only behind an explicit PRD amendment. That is overturned. Ignored ambient speech is SILENT by default. Un-addressed chatter that was never a command attempt makes no sound. Of course it shouldn't — acking every overheard sentence makes the product unbearable (PRD §1 / REQ-9 restraint). AC6.4 is read to require distinct acks only for addressed outcomes; ambient pass = silence is the requirement, not a knob.

The decision

Acks exist for explicit state transitions (wake, spawn, mute, halt, etc.) and for acknowledging addressed commands. Ignored / un-addressed ambient speech (observe.pass / route.pass) is silent — a positive "no sound," not a near-subliminal tick. There is no amendment gate and no ambientPassAck knob: silence for ignored ambient is simply how the product behaves.

CueDecision.addressed (set deterministically — was a wake/callsign/command token present? — never by the LLM) exists so an addressed near-miss gets a declined-tick while genuinely ignored ambient speech stays silent. Acking the addressed near-miss is useful feedback ("I heard you but did nothing"); acking ambient chatter is noise.

No knob, no amendment gate

ambientPassAck: SILENT   (fixed)

Example

"…so then I refactored the parser…"            → ambient pass   → SILENCE
"Atlasss, make it faster"  (mis-heard callsign) → addressed pass → declined tick
"Atlas, make the header blue"                   → route.steer    → tick-tick
"Here's an idea: …"  (engine fired)             → route.suggestion → whoosh

Verification

  • Unit: ignored ambient → silence (positive assertion: no ack emitted); addressed near-miss → declined tick; steer → tick-tick; suggestion → whoosh, all pairwise distinct. RBG: make ambient pass emit any tick → ambient-silence test fails.
  • E2e: a stream of un-addressed chatter produces zero acks; an addressed near-miss emits the declined tick; the addressed routes are audibly distinct.
