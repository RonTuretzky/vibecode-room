The contradiction (R6) and the second-round ruling (R14)

R6: "AC6.4 says each routed utterance must produce a distinct audible ack distinguishing fed-the-idea-engine, steering-process-X, and ignored observe.pass; the design said route.pass is silence." · R14: "The product rationale for silence is strong, but the engineering doc cannot be approved while intentionally violating a literal AC. Amend AC6.4 first or implement the current requirement."

Updated decision: we now implement AC6.4 as written. Every routed utterance — including ambient observe.pass — gets a distinct audible ack by default, so the doc no longer knowingly violates a literal AC. The restraint-motivated silence survives only as a live knob unlocked solely by an explicit PRD amendment.

The four distinct Layer-B acks (default = AC6.4-compliant)

RouteCaseAck (Layer B, non-tonal)
route.suggestionfed the idea enginesoft "whoosh"
route.steer:Xsteered process Xdouble-click "tick-tick"
route.pass addressedcommand attempt that resolved to no-op: near-miss, low-confidence, dropped steerdeclined tick (single low blip)
route.pass ambientun-addressed chatter, never a command attemptnear-subliminal soft tick — the minimal distinct ack AC6.4 requires

All four are pairwise distinct. CueDecision.addressed (set deterministically — was a wake/callsign/command token present? — never by the LLM) exists only so the two pass cases get acoustically distinct acks, not so ambient pass is silenced.

The silence knob — gated by amendment, not shippable unilaterally

ambientPassAck: 'tick' | 'silent'
   default            = 'tick'      (AC6.4-compliant; the only setting allowed today)
   'silent'  REQUIRES   prdAmendment: 'AC6.4-silence'   ← config-load REFUSES silence without it

So code cannot select silence until the PRD says so. The build is compliant today; the product's preferred restraint is one flag away iff the PRD amends.

Example

"…so then I refactored the parser…"          → ambient pass   → near-subliminal tick   (default)
"Atlasss, make it faster"  (mis-heard callsign)→ addressed pass → declined tick
"Atlas, make the header blue"                  → route.steer    → tick-tick
"Here's an idea: …"  (engine fired)            → route.suggestion → whoosh

Process recommendation (non-blocking — the build does NOT depend on it)

We still recommend the PRD scope AC6.4's distinct-ack to addressed utterances and declare silence the intended ack for ambient observe.pass (consistent with REQ-9 restraint / PRD §1's "over-speaking makes the product unbearable"). If adopted, set ambientPassAck:'silent'. Unlike the prior draft, the doc is approvable as-is because the default honors the current AC.

Verification

  • E2e: each ambient pass emits the near-subliminal tick by default; an addressed near-miss emits the declined tick; the four routes are audibly distinct.
