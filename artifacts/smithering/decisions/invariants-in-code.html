Routing & safety authority lives in code, not the LLM

ENG-A-04   Panopticon V0 engineering  ·  the determinism boundary  ·  docs/planning/03-eng.md §4, §7

The problem

The product spawns code-writing agents from spoken commands rolling on ~7.4% WER ASR. A misheard sentence that triggers a destructive action is a real safety hazard (PRD R-Mistranscription). If the decisions that authorize actions — "does this utterance steer a process?", "is this a destructive verb?", "does mute out-rank panic?" — were made by the LLM, they would be non-deterministic, untestable for exact behavior, and unauditable. We cannot hold a 10×–100× validation bar over a surface whose answers change run to run.

Decision

Split the two responsibilities cleanly:
The LLM doesDeterministic code does
Rolescores quality & intent (is this buildable? how confident?)holds authority — what is allowed to happen
Examplessuggestion eligibility, MCQ generation, 15-word summarizationpriority ladder, routing exclusivity, "no steer without callsign", destructive-verb classification, dead-man timer
Propertynon-deterministic (tested for shape/invariants)deterministic (tested for exact behavior, red-before-green)
The action payload is never self-authorizing. Destructive verbs are caught by a static whitelist at the point the action is produced, before execution — not by trusting the LLM's classification.

What this makes testable

priority ladder      mute > panic > stop > steer > suggest > pass   (deterministic comparator)
dispatch invariant   steering verb + no callsign + no open window  → REJECTED at dispatch (REQ-6 AC6.1)
routing exclusivity  each utterance → exactly one of {suggestion, steer:X, pass}
safety gate          verb ∈ {delete,overwrite,force-push,rm,drop,truncate} → read-back + 25 s timer
determinism          same transcript → same routing decision, every replay (REQ-7 AC7.3)

Alternatives

LLM decides routing & safety (agent-mediated). Rejected: non-deterministic authority over destructive actions; also explicitly out of V0 scope (NG-2). A misheard sentence could authorize a delete.
Hybrid — LLM proposes, code rubber-stamps. Rejected: if code can override, code is the authority anyway; if it can't, the LLM is. Make the split explicit instead.
LLM scores; code authorizes (chosen). Chosen. The non-deterministic surface is fenced to quality/intent; everything that can hurt is deterministic and tested for exact behavior.

Verification

  • Unit: priority-ladder (RBG: demote mute → fails); dispatch-invariant (RBG: remove guard → un-addressed talk steers → fails); tier-gating ("Yes" outside the suggestion window is inert); collision-guard; determinism (replay N× → identical).
  • Unit: safety posture (destructive verb → read-back + wait; no confirm in 25 s → abort; RBG: fire without confirm → fails); fuzz (garbled/double/wrong-process confirm → never double-execute).
  • E2e: un-addressed talk never steers; near-homophone of a callsign never mis-selects; destructive act blocks until "confirm".
Boundary rule: on AI-output surfaces assert shape/invariants ("≤3 MCQs", "≤15 words"), never exact text. On authority surfaces assert exact deterministic behavior. The two are tested differently on purpose.
