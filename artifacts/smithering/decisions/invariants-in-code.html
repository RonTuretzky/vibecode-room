Routing authority lives in code, not the LLM

ENG-A-04   Vibersyn V0 engineering  ·  the determinism boundary  ·  docs/planning/03-eng.md §4, §7

The problem

The product spawns code-writing agents from spoken commands rolling on ~7.4% WER ASR. If the decisions that authorize routing — "does this utterance steer a process?", "does mute out-rank panic?" — were made by the LLM, they would be non-deterministic, untestable for exact behavior, and unauditable. We cannot hold our validation bar over a surface whose answers change run to run.

Decision — CONFIRMED, split kept

Confirmed. Split the two responsibilities cleanly:
The LLM doesDeterministic code does
Rolescores quality & intent (is this buildable? how confident?)holds routing authority — where an utterance goes
Examplessuggestion eligibility, MCQ generation, 15-word summarizationpriority ladder, routing exclusivity, "no steer without callsign"
Propertynon-deterministic (tested for shape/invariants)deterministic (tested for exact behavior, red-before-green)
Routing authority is never self-authorizing by the LLM. The LLM's score is advisory; deterministic code decides the route.

Note on safety: "safety authority" matters less now — we run dangerously / run to completion (ENG-A-08/11/12 are cut: no destructive read-back gate, no dead-man timer, no shell classifier; sandbox later, not permission gating). But routing-in-code stays exactly as confirmed here.

What this makes testable

priority ladder      mute > panic > stop > steer > suggest > pass   (deterministic comparator)
dispatch invariant   steering verb + no callsign + no open window  → REJECTED at dispatch (REQ-6 AC6.1)
routing exclusivity  each utterance → exactly one of {suggestion, steer:X, pass}
determinism          same transcript → same routing decision, every replay (REQ-7 AC7.3)

Alternatives

LLM decides routing (agent-mediated). Rejected: non-deterministic routing authority; also explicitly out of V0 scope (NG-2). A misheard sentence could mis-route.
Hybrid — LLM proposes, code rubber-stamps. Rejected: if code can override, code is the authority anyway; if it can't, the LLM is. Make the split explicit instead.
LLM scores; code routes (chosen). Chosen. The non-deterministic surface is fenced to quality/intent; routing is deterministic and tested for exact behavior.

Verification

  • Unit: priority-ladder (RBG: demote mute → fails); dispatch-invariant (RBG: remove guard → un-addressed talk steers → fails); collision-guard; determinism (replay N× → identical).
  • E2e: un-addressed talk never steers; near-homophone of a callsign never mis-selects.
Boundary rule: on AI-output surfaces assert shape/invariants ("≤3 MCQs", "≤15 words"), never exact text. On routing-authority surfaces assert exact deterministic behavior. The two are tested differently on purpose.
